#######################################################################
# 第1章 培养Pythonic思维

# 第1条  查询自己使用的Python版本
# (1) Python3是最新版的Python,应该用Python3开发项目
# (2) 在操作系统的命令行界面运行Python时，确认该Python的版本是否跟你要使用的版本相同
$ python  --version

# (3) 不要再用Python2做开发了，因为该版本已于2020年1月1日停止更新维护

# 第2条  遵循PEP8风格指南
# (0) 编写Python代码时，总是应该遵循PEP8风格指南
#     与广大Python开发者采用同一套代码风格，可以使项目更利于多人协作
#     采用一致的风格编写代码，代码的后续修改更容易
# (1) 与空白有关的建议
#  1) 用空格表示缩进而非制表符;每层缩进用4个空格;每行不超过79个字符
#  2) 占据多行的长表达式，除首行外的其余各行都应该在通常的缩进级别之上再加4个空格
#  3) 同一文件中，函数与类之间用两个空行隔开;同一类中，方法与方法之间用一个空行隔开
#  4) 字典中，键与冒号间不加空格，冒号与值之间加一个空格
#  5) 变量赋值时符号左右各加一个空格;变量类型注解中,变量名和冒号间无空格,类型信息前有
# (2) 与命名有关的建议
#  1) 函数、变量及属性用小写字母来拼写，各单词之间用下划线相连 my_app
#  2) 受保护实例属性开头一个下划线;私有实例属性用两个;类或异常命名时单词首字母大写
#  3) 模块级别的常量，所有字母都大写，各单词之间用下划线相连 ALL_CAPS.
#  4) 实例方法第一个参数命名为se1f;类方法的第一个参数命名为c1s

# 第3条  了解bytes与str的区别
# 第4条  用支持插值的f-string取代C风格的格式字符串与str.format方法
# 第5条  用辅助函数取代复杂的表达式
# 第6条  把数据结构直接拆分到多个变量里，不要专门通过下标访问
# 第7条  尽量用enumerate取代range
# 第8条  用zp函数同时遍历两个迭代器
# 第9条  不要在for与while循环后面写else块
# 第10条 用赋值表达式减少重复代码
